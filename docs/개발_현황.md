# 개발 현황 문서

작성일: 2026-04-19

## 1) 프로젝트 현재 상태

- 서비스명: `youtube` (Next.js App Router 기반)
- 배포 브랜치: `main`
- 정산 기능 상태: 운영 가능한 수준으로 완료
- 데이터 저장: Upstash Redis 우선, 실패 시 메모리/로컬 백업

## 2) 구현 완료 범위

### 화면(페이지)

- `/login`: 로그인
- `/admin`: 방송 운영/정산 관리 대시보드
- `/settlements`: 정산 기록 목록/검색/대시보드/삭제 로그
- `/settlements/[id]`: 정산 상세 및 내보내기
- `/overlay`: OBS/Prism 오버레이
- `/create-accounts`: 계정 생성/관리
- `/alerts`, `/youtube`, `/simple-admin`, `/test-realtime`: 운영/테스트 보조 화면

### 정산 기능

- 정산 계산(계좌/투네 비율 + 세금 반영)
- 멤버별 정산 결과 집계
- 정산 기록 저장/조회/삭제
- 정산 기록 JSON 백업/복구
- 정산 데이터 3초 주기 동기화 + 탭 포커스 동기화
- 월별/멤버별/후원자 기반 대시보드 시각화

### API

- 인증: `/api/auth/login`, `/api/auth/me`, `/api/auth/logout`
- 상태: `/api/state` (조회/저장)
- 정산: `/api/settlements` (조회/저장)
- 기타: `/api/events`, `/api/websocket`, `/api/accounts`, `/api/daily-log`, `/api/overlays`

## 3) 최근 안정화(커밋 기준)

- 레거시 Redis 데이터 마이그레이션 대상 계정 제한 처리
- 다중 탭/브라우저 저장 시 후원 목록 병합으로 덮어쓰기 방지
- 정산 기록 디바이스 간 동기화(서버 source of truth) 적용
- 서버 재시작/빈 응답 상황에서 로컬 데이터 보존 강화
- 미션/멤버/후원 데이터 의도치 않은 초기화 방지 로직 보강
- 오버레이(Prism) 확대/축소 관련 이슈 보정 및 회귀 대응

## 4) 현재 작업 트리 참고사항

- 로컬에 미추적 파일이 존재하며, 아직 커밋/배포 대상 확정 전 상태
  - `docs/테스트_가이드.md`
  - `tests/test-settlement-ratio.mjs`
  - `src/app/test-settlement/`

## 5) 다음 단계(추가 기능 개발 전 체크)

- 추가 기능 요구사항 확정: 대상 화면, 입력/출력, 권한, 완료 기준
- 데이터 모델 영향 확인: `AppState`, 정산 레코드 스키마 영향 범위 점검
- 동기화 정책 유지: 멀티탭/멀티디바이스 병합 규칙과 충돌 없는지 검토
- 배포 전 검증: 로컬 테스트 + Render 배포 로그 + 핵심 API 헬스체크
