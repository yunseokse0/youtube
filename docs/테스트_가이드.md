# 테스트 가이드

## 1. 계정 생성 사이트 테스트

### 실행 방법
```bash
npm run dev
```
→ http://localhost:3000 에서 접속

### 계정 생성 테스트 절차
1. **접속**: http://localhost:3000/create-accounts
2. **접근 키 입력**: `.env`에 설정된 `ADMIN_ACCOUNTS_KEY` 입력 (기본값: `finalent`)
3. **계정 생성**:
   - 이름, 회사명, 비밀번호 입력
   - 무제한 또는 기간 설정
   - 생성 버튼 클릭
4. **로그인**: http://localhost:3000/login 에서 생성한 계정으로 로그인

### 환경변수 확인
- `.env` 또는 `.env.local`에 `ADMIN_ACCOUNTS_KEY=finalent` 설정 필요
- 없으면 `.env.example` 참고하여 생성

## 2. 멤버별 정산 비율 테스트

### 테스트 시나리오
각 멤버마다 **계좌 비율**, **투네 비율**이 다를 때 정산이 올바르게 계산되는지 확인

### 테스트 절차

#### Step 1: 관리자 페이지 접속
- 로그인 후 http://localhost:3000/admin

#### Step 2: 멤버별 금액 입력
예시 데이터:
| 멤버 | 계좌 | 투네 |
|------|------|------|
| 멤버1 | 100,000 | 50,000 |
| 멤버2 | 80,000 | 40,000 |
| 멤버3 | 60,000 | 30,000 |
| 운영비 | 20,000 | 10,000 |

#### Step 3: 멤버별 개별 비율 설정
- **방송 종료 정산** 섹션에서 "멤버별 개별 비율 사용" 체크
- 각 멤버별로 계좌 비율 %, 투네 비율 % 입력

**테스트 케이스 A** (멤버마다 다른 비율):
| 멤버 | 계좌 비율 | 투네 비율 | 예상 계산 |
|------|----------|----------|-----------|
| 멤버1 | 70% | 60% | (10만×0.7 + 5만×0.6) - 세금 |
| 멤버2 | 80% | 50% | (8만×0.8 + 4만×0.5) - 세금 |
| 멤버3 | 60% | 70% | (6만×0.6 + 3만×0.7) - 세금 |
| 운영비 | 100% | 100% | (2만+1만) 세금 없음 |

**계산식**: `(계좌×계좌비율 + 투네×투네비율) × (1 - 세금비율)`

#### Step 4: 정산 생성
- 정산 제목 입력 (예: "비율 테스트 정산")
- "방송 종료(정산 생성)" 클릭

#### Step 5: 결과 확인
- 정산 상세 페이지에서 각 멤버의 **적용 비율**, **매출**, **세금**, **최종 정산** 확인
- 운영비 멤버는 세금 0%, 비율 100% 적용되는지 확인

### 검증 포인트
- [ ] 멤버1: 계좌 70%, 투네 60% 적용
- [ ] 멤버2: 계좌 80%, 투네 50% 적용
- [ ] 멤버3: 계좌 60%, 투네 70% 적용
- [ ] 운영비: 100% 적용, 세금 0
- [ ] 총 정산 합계 = 각 멤버 net 합계

## 5. 문제 발생 시

- **계정 생성 401**: `ADMIN_ACCOUNTS_KEY` 환경변수 확인
- **로그인 실패**: 계정 생성 시 설정한 비밀번호 확인
- **정산 비율 이상**: "멤버별 개별 비율 사용" 체크 여부, % 입력값(0~100) 확인
