# Excel Broadcast Settlement 시스템 소개서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **패키지명** | `excel-broadcast-settlement` |
| **버전** | 0.1.0 |
| **설명** | Final Entertainment 방송 정산 관리자 및 OBS/Prism 오버레이 시스템 |
| **용도** | 유튜브/방송 후원 정산, 멤버별 금액 관리, 방송 오버레이 표시 |

방송 중 후원(계좌/투네)을 실시간으로 관리하고, 방송 종료 시 정산을 생성·저장·내보내기하는 웹 기반 시스템입니다.

## 4. 핵심 기능 상세

### 4.1 Admin (관리자)

- **대시보드**: 오늘 총 후원액, 후원 건수, 멤버 수
- **멤버 관리**: 추가/삭제/이름 변경, 계좌·투네 금액 입력, 개인 목표, 운영비 멤버
- **후원자 기록**: 후원자명, 금액, 대상(계좌/투네), 멤버 연결, 건별 기록
- **후원자별 누적**: 후원자별 계좌/투네/총합, 건수
- **채팅용 복사**: 정산 요약 텍스트 생성 및 복사
- **미션 메뉴판**: 미션 추가/수정/삭제, 순서 변경, HOT 표시
- **오버레이 설정**: 다중 프리셋, 테마, 배율, 표시 요소 ON/OFF
- **방송 종료 정산**: 계좌/투네/세금 비율, 멤버별 비율, 정산 생성
- **데이터 관리**: 상태 JSON 내보내기, 일일 로그, 후원자 CSV, 정산 리셋

### 4.2 Overlay (방송 오버레이)

- **OBS/Prism 브라우저 소스**: `/overlay?p={presetId}&u={userId}` 형태 URL
- **표시 요소**: 멤버 목록, 총합, 목표바, 개인 골, 후원 티커, 타이머, 미션 메뉴
- **테마**: default, excel, neon, neonExcel, retro, minimal, rpg, pastel
- **실시간 동기화**: API/localStorage 폴링(약 700ms), Storage 이벤트
- **데모 모드**: `?demo=true`로 샘플 데이터 표시

### 4.3 Settlements (정산)

- **정산 기록**: 최대 3년 보관
- **검색**: 제목, 출연자, 날짜
- **상세**: 닉네임, 실명, 은행, 계좌번호, 예금주, 계산식
- **내보내기**: TXT(메모장), CSV, PDF
- **멤버 순서**: 정산금액 내림차순, 운영비 맨 아래
- **계산식**: (계좌×계좌비율 + 투네×투네비율) - 세금비율

## 5. API 구조

| 메서드 | 경로 | 역할 |
|--------|------|------|
| POST | `/api/auth/login` | 로그인 (id, password) |
| - | `/api/auth/me` | 현재 사용자 조회 |
| POST | `/api/auth/logout` | 로그아웃 |
| GET | `/api/state` | 앱 상태 조회 (members, donors, overlayPresets 등) |
| POST | `/api/state` | 앱 상태 저장 |
| GET | `/api/settlements` | 정산 기록 목록 조회 |
| POST | `/api/settlements` | 정산 기록 저장 |
| - | `/api/events` | SSE 이벤트 스트림 |
| - | `/api/websocket` | WebSocket |

**저장소**: Upstash Redis (`UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`). 미설정 시 메모리 사용.

## 6. 데이터 흐름

### 6.1 상태 관리

```
AppState = {
  members: Member[];      // 멤버 목록 (계좌, 투네, 목표, 직급)
  donors: Donor[];       // 후원 기록
  forbiddenWords: string[];
  missions?: MissionItem[];
  overlayPresets?: OverlayPreset[];
  updatedAt: number;
}
```

### 6.2 동기화 흐름

```
[Admin 페이지]
    │
    ├─ loadStateFromApi() ──► GET /api/state (Redis)
    ├─ saveStateAsync() ───► POST /api/state
    ├─ localStorage (STORAGE_KEY) ──► 로컬 캐시
    └─ Storage 이벤트 ──► 다른 탭 변경 감지

[Overlay 페이지]
    │
    ├─ useRemoteState() ──► loadStateFromApi() (700ms 폴링)
    ├─ localStorage ──► 동일 탭 변경 즉시 반영
    └─ Storage 이벤트 ──► Admin 탭 변경 감지
```

### 6.3 정산 동기화

```
[Settlement]
    │
    ├─ loadSettlementRecordsPreferApi() ──► GET /api/settlements
    ├─ saveSettlementRecordsToApi() ───► POST /api/settlements
    └─ localStorage (SETTLEMENT_RECORDS_KEY) ──► 로컬 백업
```

### 6.4 인증

- **쿠키**: `sb_user` (30일, httpOnly)
- **보호 경로**: `/admin`, `/settlements` (미인증 시 `/login` 리다이렉트)
- **기본 계정**: `finalent` / `finalent`
