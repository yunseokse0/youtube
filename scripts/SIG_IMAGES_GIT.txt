시그 이미지 · GitHub · Render (요약)
====================================

■ 완판 도장(STAMP) — 시그 판매 완료 오버레이
  경로: public/images/sigs/stamp.svg  (기본값)
  「기본 도장 사용」= /images/sigs/stamp.svg. 다른 파일은 관리자 → 완판 도장 → 「저장소」.

■ 신규 롤링 시그 GIF (추가만, 기존 파일 덮어쓰기 안 함)
  1) PC 폴더 → 프로젝트 (신규 파일만):
       npm run sig:import:new -- "D:\Downloads\siggif"
     → public/images/sigs/from-drive/  (이미 있으면 건너뜀)

  2) Git (추가된 파일만 add 권장):
       git add public/images/sigs/from-drive/새파일.gif
       git commit -m "chore: 시그 GIF 추가"
       git push

  3) Render 관리자 → 시그 롤링:
       「GitHub 신규만 롤링 추가」 버튼 (목록에 없는 /images/sigs/… 만 추가)
       또는 「저장소에서 일괄 추가」

  OBS는 GIF를 raw.githubusercontent.com 에서 직접 로드(캐시).
  변경된 목록만 /api/state 로 동기화(SSE+304).

■ 환경 변수 (Render, Supabase 없음)
  NEXT_PUBLIC_SIG_IMAGES_GITHUB_ONLY=true
  NEXT_PUBLIC_SIG_ROLLING_GITHUB_BASE=https://raw.githubusercontent.com/OWNER/REPO/main/public

■ 전체 덮어쓰기가 필요할 때만
  npm run sig:import -- "D:\folder"   (--new-only 없음)

참고: NEXT_PUBLIC_SIG_IMAGES_PLACEHOLDER_ONLY=1 이면 /images/sigs/… 가 더미로 바뀌므로 끄세요.
