시그 GIF·완판 스탬프를 Git으로 배포에 포함하는 방법 (요약)
====================================================

1) Google Drive에서 폴더 전체 ZIP 다운로드 후 압축 해제

2) 프로젝트 루트에서 복사 스크립트 실행 (경로는 본인 PC 기준으로 바꿈)

   · 기본 대상 폴더(from-drive, 구글 드라이브 등 범용):

     npm run sig:import -- "D:\Downloads\시그폴더"

     → public/images/sigs/from-drive/ 아래에 .gif 가 복사됩니다.

   · 추가 시그 GIF 전용 폴더(siggif):

     npm run sig:import:siggif -- "D:\Downloads\siggif"

     → public/images/sigs/siggif/ 아래에 .gif 가 복사됩니다.

   · 대상 하위 폴더를 직접 지정:

     node scripts/copy-sigs-from-folder.mjs "D:\Downloads\foo" myfolder

     → public/images/sigs/myfolder/

3) Git에 올리기

   git add public/images/sigs/from-drive public/images/sigs/siggif .gitattributes scripts/
   git add public/img/sig-sold-stamp.svg
   git status
   git commit -m "chore: 시그 GIF·완판 스탬프 번들"
   git push

4) 관리자(시그 판매)에서 각 시그의 이미지 URL 예시

   /images/sigs/from-drive/파일이름.gif
   /images/sigs/siggif/파일이름.gif

   (배포 Origin 기준 동일 오리진 경로)

5) 완판 스탬프 이미지

   기본 경로: public/img/sig-sold-stamp.svg  → URL /img/sig-sold-stamp.svg

   레거시: public/images/sigs/stamp.svg 도 동작할 수 있습니다.

6) 용량이 많을 때

   - 저장소·clone 크기가 커지므로, GIF가 매우 많으면 Git LFS 사용을 검토하세요.
   - 또는 Supabase Storage 등 외부 저장소 + 업로드 API 경로를 유지하는 편이 나을 수 있습니다.

참고: NEXT_PUBLIC_SIG_IMAGES_PLACEHOLDER_ONLY=1 이면 커스텀 /images/sigs/… 가 더미로 바뀌므로 끄세요.

오버레이 동기화: 기본은 주기 GET 없음(SSE state_updated·storage). 폴링이 필요하면 URL에 overlayPollMs=3000 등(700~120000 ms).
